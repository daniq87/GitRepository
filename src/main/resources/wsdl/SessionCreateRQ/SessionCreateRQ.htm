Description

The SessionCreateRQ service is required to open connections with Sabre Web Services . SessionCreateRQ explicitly creates a Sabre Web Services connection. When your client sends a correctly-formatted SessionCreateRQ SOAP message with your security credentials to any of the designated endpoints, the Sabre Web Services gateway does the following:

    • Receives the request
    • Authenticates the request and creates a connection
    • Authorizes the client to access specific Sabre Web Services and other business applications or systems within Sabre Holdings, based on your security credentials
    • For user IDs and Web services that require a Sabre "host" session, allocates a Sabre session from your session pool (A Sabre session is another name for a TA, and a session pool is also referred to as a TAM pool. This documentation uses the terms Sabre session and session pool.)
    • Returns a unique, encrypted security token to the client in the SessionCreateRS response message, in <wsse:BinarySecurityToken> , a child of <wsse:Security>
    • Returns the same conversation ID in eb:ConversationId , from the SessionCreateRQ message
    • Returns a reference to the message ID of the corresponding request in <eb:RefToMessageId>

The conversation ID and security token in the returned SessionCreateRS message are referred to as the connection ID. Their return means the connection to the Sabre Web Services gateway has been created and a Sabre session is allocated.

The client must extract and store the returned values for eb:CPAId , eb:ConversationId, and the entire wsse:security@wsse:BinarySecurityToken node for inclusion in subsequent requests and workflows that use this connection.

When sending Web service requests for travel content, the client includes the connection ID and the same value for eb:CPAId in all transactions that use this connection, whether the client maintains state or not.

You can connect to the production URL or a URL representing one of the certification or development systems.

When you no longer need the connection, it is required to send the SessionCloseRQ Service with the same connection ID to close the connection. This properly cleans up the session and releases all resources allocated to the connection and its session.

Errors are returned in SOAP fault format, with or without ebXML headers.

Dependencies

Security credentials, consisting of a username, password, organization, and domain are required to connect. Use the values you were provided when you registered for Sabre Web Services .

Developer Notes

Service- and User-Specific Values for the SOAP Message

      1. Values for the request payload that are particular to this Web service are as follows:

    MimeHeader content ID = SessionCreateRQ

    Document root element = SessionCreateRQ

    PseudoCityCode attribute = IPCC

    For payloads included inside the SOAP envelope, eb:Reference and MimeHeader content ID do not apply.

      2. Service-specific values for the request SOAP envelope are as follows:

      • eb:CPAId = The value you were provided for Organization
      • eb:Service = Session
      • eb:type = sabreXML
      • eb:Action = SessionCreateRQ
      • eb:Reference/xlink:href = cid:SessionCreateRQ

Other Notes

      3. The maximum field size is validated by the gateway. If the field size is exceeded, an error will be returned. For more information about validated field sizes, see Appendix B in Guide to Accessing and Using Services .

      4. When your client reuses or retrieves a connection created with a particular SessionCreateRQ message, all requests must include the returned security token and the same values for eb:ConversationId , eb:CPAId , and if applicable, the Source/PseudoCityCode that were used to create the connection and session being used.

      5. Although some Sabre Web Services require you to call a specific version number in the request payload, do not request a service version in the session management messages. While the SessionCreateRQ payload returns a version in the response payload, this version is not related to other travel-related Web services that retrieve business or travel content.

      6. The request payload does not include an xmlns attribute with the document root element in the SessionCreateRQ message. This attribute is, however, returned in the payload of the SessionCreateRS response .
