Description

This service explicitly closes a Sabre Web Services connection. Consuming this service is required to close connections because it properly cleans up the session and releases all resources allocated to the connection and its session.

When your client sends a correctly-formatted SessionCloseRQ SOAP message with the security token and connection ID from the connection it wants to close, the Sabre Web Services gateway does the following:

    • Closes the Sabre Web Services connection
    • If applicable, terminates the Sabre session that was allocated to the connection. This returns your session (formerly known as a TA) to your session pool (formerly called a TAM pool).
    • Discards content in the Sabre work area (also known as the AAA)
    • Invalidates the security token
    • Returns the namespace attribute ( xmlns ) in the document root element of the response payload

Errors are returned in SOAP fault format, with or without ebXML headers.

Dependencies

A live connection to Sabre Web Services , and the binary security token, conversation ID, eb:CPAId , and PseudoCityCode of the connection to be closed.

Developer Notes

Service-Specific Values for the SOAP Message

      1. Values for the request payload that are particular to this Web service are as follows:

    MimeHeader content ID = SessionCloseRQ

    Document root element = SessionCloseRQ

    PseudoCityCode attribute = IPCC

    For payloads included inside the SOAP envelope, eb:Reference and MimeHeader content ID do not apply.

      2. Service-specific values for the request SOAP envelope are as follows:

      • eb:CPAId = The value you were provided for Organization , that you used to open the connection
      • eb:Service = Session
      • eb:type = sabreXML
      • eb:Action = SessionCloseRQ
      • eb:Reference/xlink:href = cid:SessionCloseRQ

Other Notes

      3. The maximum field size is validated by the gateway. If the field size is exceeded, an error will be returned. For more information, see Appendix B in Guide to Accessing and Using Services .

      4. Although some Sabre Web Services require you to call a specific version number in the request payload, do not request a service version in the session management messages. While the SessionCloseRS payload returns a version in the response payload, this version is not related to other travel-related Web Services that retrieve business or travel content.

      5. The value for wsse:BinarySecurityToken that was returned in SessionCreateRS of the connection being closed must be passed in the SessionCloseRQ request.

      6. Do not send an xmlns attribute with the document root element in the payload of the SessionCloseRS message. This is unlike some other Sabre Web Services , where the xmlns attribute is required. This attribute is, however, returned in the payload of the SessionCreateRS response .

      7. The values for eb:CPAId , Organization , and Source/PseudoCityCode must match the values sent in the SessionCreateRQ request message that was used to create the connection being closed.
